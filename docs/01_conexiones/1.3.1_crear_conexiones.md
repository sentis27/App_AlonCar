1. ID de Tarea y Nombre
Tarea 1.3: Crear Conexiones, Contraseñas y Accesos (Configuración MCP y Antigravity).

2. Flujo de Información (Arquitectura de Accesos)
El sistema opera bajo un modelo de confianza cero (Zero Trust) en el código fuente, garantizando que ninguna credencial quede expuesta:

Origen Maestro: Las credenciales, tokens y accesos se gestionan y listan de forma centralizada y encriptada en una planilla de Google Sheets.

Inyección Local: Los datos extraídos se inyectan en la máquina de desarrollo de manera local a través de un archivo .env. Este archivo aísla las claves de n8n, Google y GitHub para que nunca se filtren a internet.


Capa de Protección: El archivo .env y el JSON de credenciales de Google están estrictamente listados en el .gitignore.

Consumo (MCP/Antigravity): El servidor MCP lee las variables de entorno locales de forma segura para autenticarse contra el n8n local y las APIs de Google Workspace sin exponer el texto plano al agente de IA.

3. Estructura Recomendada del Archivo .env
El archivo .env debe ubicarse en la raíz del proyecto y contener la siguiente estructura obligatoria:

Fragmento de código
# CONFIGURACIÓN GENERAL DEL PROYECTO
ENV=development

# LOCAL n8n CONFIGURATION
N8N_API_URL=http://localhost:5678/api/v1
N8N_API_KEY=TU_API_KEY_DE_N8N_AQUI

# GITHUB CONFIGURATION
GITHUB_TOKEN=TU_TOKEN_DE_GITHUB_PERSONAL_AQUI

# GOOGLE CREDENTIALS PATH
GOOGLE_APPLICATION_CREDENTIALS=./service-account.json
4. Protocolo de Validación
Para confirmar que la conexión es segura y operativa, ejecutar el siguiente checklist:

Auditoría de Git: Ejecutar git status. El archivo .env y service-account.json no deben aparecer en la lista de archivos rastreados (Untracked files).

Prueba de Conectividad n8n: Enviar un comando simple desde Antigravity a través del MCP solicitando listar los flujos activos en el localhost:5678.

Prueba de Google Cloud: Solicitar al MCP que lea los encabezados de la planilla de Google Sheets maestra utilizando los permisos de la Service Account inyectada.
